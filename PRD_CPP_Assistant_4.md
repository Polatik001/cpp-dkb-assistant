# PRD — AI-ассистент начальника ЦПП ДКБ
**Продукт:** Рабочий AI-ассистент начальника отдела  
**Организация:** АО «Казахтелеком» · Филиал · ДКБ ЦПП  
**Версия:** 1.0  
**Дата:** 31 мая 2026  
**Статус:** Draft  

---

## 1. Обзор продукта

AI-ассистент для начальника Центра продаж и поддержки (ЦПП) Дирекции корпоративного бизнеса (ДКБ). Инструмент объединяет контроль KPI, управление 33 менеджерами, планирование и генерацию рабочих документов в одном интерфейсе.

### Проблема

Начальник ЦПП ежедневно работает с большим объёмом данных: показатели продаж по регионам, задачи 33 менеджеров, входящие заявки, отчётность для руководства. Эти процессы разрознены, требуют ручного сбора информации и занимают значительное рабочее время.

### Решение

Единый AI-ассистент, который агрегирует данные, ведёт чеклисты планёрок, контролирует эффективность сотрудников и генерирует документы через встроенный AI-чат.

## 3. Цели продукта

- Сократить время на ручной сбор данных по KPI и менеджерам
- Централизовать контроль задач 33 менеджеров
- Автоматизировать подготовку рабочих документов и отчётов
- Обеспечить структурированное планирование (день / неделя / месяц)

## 4. Функциональные требования

### 4.1 Дашборд KPI

**Описание:** Главный экран с ключевыми показателями месяца и прогрессом по регионам.

**Показатели:**
- Продажи (факт / план, млн ₸)
- Выручка (факт / план, млн ₸)
- Обработанные заявки (факт / план, шт.)
- Клиентская база (факт / план, клиентов)

**Регионы:** Алматы, Астана, Шымкент, Қарағанды, Атырау — прогресс-бар % выполнения плана.

**Обновление данных:** Ручной ввод факта по каждому показателю через форму.

**Индикаторы:**
- ≥ 90% — зелёный (↑)
- 70–89% — жёлтый (→)
- < 70% — красный (↓)

---

### 4.2 Контроль сотрудников

**Описание:** Таблица всех 33 менеджеров с возможностью фильтрации и детального просмотра.

**Список менеджеров содержит:**
- Порядковый номер
- Имя
- Регион
- Количество задач
- % выполненных задач
- Уровень эффективности: Высокая / Средняя / Низкая

**Фильтры:**
- Поиск по имени
- Фильтр по эффективности

**Карточка сотрудника (детали):**
- Эффективность
- % выполнения задач
- Количество задач всего
- Количество просроченных задач
- Список задач со статусами: В работе / Выполнено / Просрочено
- Кнопка добавления новой задачи сотруднику
- Кнопка AI-анализа сотрудника (генерирует рекомендации)

**Статистика вверху:**
- Всего сотрудников: 33
- Эффективных (высокая)
- Проблемных (низкая)

---

### 4.3 Планёрка

**Описание:** Чеклисты для ежедневной и еженедельной планёрки.

**Ежедневный чеклист (7 пунктов):**
1. Проверить выполнение вчерашних поручений
2. Просмотреть сводку продаж за вчера
3. Проверить входящие заявки и очередь
4. Провести планёрку с командой (10–15 мин)
5. Зафиксировать поручения с дедлайнами
6. Отметить критические отклонения от плана
7. Направить оперативный отчёт руководству

**Еженедельный чеклист (8 пунктов):**
1. Анализ KPI по всем регионам
2. Сравнение план/факт по продажам и выручке
3. Контроль динамики клиентской базы
4. Анализ скорости обработки заявок (SLA)
5. Оценка эффективности менеджеров
6. Совещание по итогам недели
7. Корректировка задач на следующую неделю
8. Подготовка еженедельного отчёта

**Функции:** Отметка выполненного, сброс чеклиста, генерация повестки через AI.

---

### 4.4 Еженедельный план

**Описание:** Интерактивный план по дням недели с чекбоксами.

**Дни:** Понедельник — Пятница

**Предустановленные задачи по дням:**

| День | Задачи |
|---|---|
| Понедельник | Планёрка с командой, Анализ KPI за прошлую неделю, Контроль входящих заявок |
| Вторник | Контроль плана продаж по регионам, Работа с проблемными менеджерами |
| Среда | Анализ клиентской базы, Контроль скорости обработки заявок |
| Четверг | Контроль выручки по регионам, Подготовка промежуточного отчёта |
| Пятница | Итоговая планёрка недели, Подготовка отчёта за неделю, Постановка задач на следующую неделю |

**Функции:** Добавление задачи в любой день, сброс недели, генерация плана через AI.

---

### 4.5 Месячный план

**Описание:** План на месяц, разбитый на 4 недели с типизацией задач.

**Типы задач:**
- Продажи (синий)
- KPI (красный)
- Сотрудники (зелёный)

**Предустановленные задачи:**

| Неделя | Задача | Тип |
|---|---|---|
| Неделя 1 | Постановка задач менеджерам на месяц | Сотрудники |
| Неделя 1 | Анализ KPI за прошлый месяц | KPI |
| Неделя 1 | Утверждение плана продаж | Продажи |
| Неделя 2 | Промежуточный контроль плана продаж | Продажи |
| Неделя 2 | Анализ клиентской базы | Сотрудники |
| Неделя 2 | Контроль SLA по заявкам | KPI |
| Неделя 3 | Работа с отстающими менеджерами | Сотрудники |
| Неделя 3 | Контроль выручки | Продажи |
| Неделя 3 | Корректировка задач | KPI |
| Неделя 4 | Подведение итогов месяца | KPI |
| Неделя 4 | Подготовка месячного отчёта | Продажи |
| Неделя 4 | Планирование следующего месяца | Сотрудники |

**Функции:** Добавление задачи в любую неделю, отметка выполнения, сброс месяца, генерация плана через AI.

---

### 4.6 AI-чат

**Описание:** Встроенный чат на базе Claude AI, настроенный на специфику ЦПП ДКБ.

**Быстрые шаблоны:**
- Служебная записка о выполнении плана продаж
- Поручение менеджерам по скорости обработки заявок
- Еженедельный отчёт по регионам
- Анализ неэффективных сотрудников
- Анализ выполнения плана по регионам
- План удержания клиентской базы

**Контекст AI:**
> Ассистент знает: АО «Казахтелеком», ЦПП ДКБ, 33 менеджера, задачи по KPI/продажам/выручке/клиентской базе/заявкам. Отвечает на русском языке. Даёт готовые шаблоны документов.

**Поддерживаемые документы:**
- Служебные записки
- Поручения
- Протоколы планёрок
- Еженедельные / месячные отчёты
- Планы мероприятий
- Аналитические справки

## 6. Ограничения текущей версии (v1.0)

- Данные не сохраняются между сессиями (нет базы данных)
- Имена менеджеров тестовые (Менеджер 1–33) — требуется ручная замена
- Данные по регионам вводятся вручную — нет интеграции с CRM
- Нет авторизации и разграничения прав доступа
- Нет мобильной версии
